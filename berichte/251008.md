## Optionen(shorthand, longhand)

Es gibt die kurze Schreibweise von Optionen mit einem Buchstaben diese heisen shorthand:

```sh
ls -l
````

Und natürlich auch die longhand version also die Lange.

```sh
ls --format=long
```
---

## Argumente

Argumente sind zusätzliche Optionen die einem Befehl sagen was er genau tun soll.

z.b.

```sh
 ls -l
```
dieses macht die ausgabe von ls genauer dies sieht dann so aus:

```
total 8
drwxr-xr-x 1 runner runner 100 Oct  8 06:15 berichte
-rw-r--r-- 1 runner runner 104 Sep 24 07:00 README.md
```
---

## echo

Echo ist wie ein printf gibt also einfach etwas aus

Eingabe:
```sh
echo Hallo
```

Ausgabe:
```
Hallo
```
---

## History

Die History sind die Befehle die man zuvor in der Shell eingegeben hat.
Diese lassen sich durch das Drücken von den Pfeiltasten nach oben und unten schnell wieder
aufrufen und neu verwenden. Löschhen kann man die history mit ```history -c```.

---

## Tastenkürzel 

Es gibt verschieden Tastenkürzel wie z.b.:

`Strg a` damit springt man zum Anfang eines Befehls.
`Strg e` damit an das Ende eines Befehls.
`Strg c` damit wird der Befehl gestoppt.

# Filesystem 

## Befehle

Es gibt verschieden Befehle wie z.b.:

Eingabe:
```sh
ls
```
Ausgabe:
```
250910.md  250924.md  251008.md  img  _Joszt_Maximilian.md
```
ls zeigt die verschiedenen Dateien und directories die sich in dem Bereich wo man gerade ist befinden.
ls -l zeigt dies genauer.
ls -a zeigt auch die versteckten Dateien.

Eingabe:
```sh
cd
```
Mit cd kann man sich durch die verschiedenen directories bewegen.
Mit cd. kann man einen Schritt wieder zurück gehen.

Eingabe:
```sh
pwd 
```

Ausgabe:
```
/home/runner/workspace/berichte
```

Damit wird einem der Pfad angezeigt also wo genau man sich gerade drin befindet


Mit __mkdir__ kann man neue directories erstellen diese kann man dann auch wieder mit
__rmdir__ löschen.

Genauso kann man mit __touch__ Dateien erstellen und mit __rm__ sie wieder löschen.

---

## Working directory 

das ist das Verzeichnis in dem man sich gerade befindet wo man Befehle ausführen kann und drinnen arbeiten kann.

## Relativer- Absoluterpfad

Der __AbsolutePfad__ ist der Weg vom Wurzelverzeichnis bis zum Verzeichnis oder der Datei die gerade bearbeitet wurde 

```sh
/home/runner/workspace/berichte
```

Der __Relativepfad__ ist der Pfad vom Aktuellen Arbeitsverzeichnis bis zur Datei

```sh
berichte/img/image.png
```

Der Unterschied ist also das der Absolute vom Ursprung bis zur Datei ist und der Relative nur vom aktuellen Arbeitsverzeichnis
